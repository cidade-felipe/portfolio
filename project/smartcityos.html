SmartCityOS

Sistema de gestão urbana com PostgreSQL e Python, focado em modelagem de dados, automações com triggers, auditoria e uma aplicação desktop para operação e análises.

1. Introdução

O SmartCityOS foi desenvolvido como um sistema integrado para simular e organizar operações comuns de uma cidade, com foco no que mais importa em dados: estrutura consistente, regras automatizadas e rastreabilidade.

A ideia é demonstrar domínio de modelagem relacional e SQL avançado, usando triggers para automatizar processos, além de uma interface desktop para consulta e operação.

2. Objetivos

Construir uma base relacional robusta para entidades urbanas (cidadãos, veículos, sensores, incidentes e multas) e adicionar automações e garantias via banco, reduzindo inconsistência e centralizando regras de negócio.

3. Stack e Arquitetura

O projeto foi dividido em dois blocos: banco PostgreSQL (DDL, índices, views e triggers) e uma aplicação em Python (PySide6) para navegação de dados, dashboards e consultas.

4. Banco e Automação

O núcleo do SmartCityOS é o banco. A lógica de automação usa triggers para auditoria, controle de alterações e regras como soft delete. Isso garante rastreabilidade e consistência, mesmo que o consumo venha de diferentes interfaces.

Também foram pensados índices e views para facilitar consultas recorrentes e performance em cenários de leitura.

5. Aplicação Desktop

A aplicação em PySide6 oferece uma camada visual para navegar pelas entidades, executar consultas com segurança (modo somente leitura) e exportar resultados em Excel. Isso demonstra um ciclo completo: banco bem construído e uma interface que facilita o uso.

Resultado: um sistema completo com regras no banco, auditoria e uma interface real. Próximos passos naturais seriam testes automatizados, containerização (Docker) e uma camada de API para uso web.
