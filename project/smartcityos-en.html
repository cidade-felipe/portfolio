SmartCityOS

Smart operating system for cities, focused on relational modeling, business rules in the database, auditing and a desktop application for operations and queries.

1. Context

SmartCityOS was designed to simulate a real urban operation, connecting citizens, vehicles, sensors, incidents and fines in a centralized database. The priority was to create a consistent base with automated rules inside the database and full traceability.

The project demonstrates relational modeling, advanced SQL, automation triggers and a desktop interface focused on navigation, filters and operational reports.

2. Objectives

Build a solid data core and an application ready for operational use.

  • Model urban entities with referential integrity and database-level rules.
  • Automate critical processes (fines, payments, auditing, soft delete).
  • Provide screens with KPIs, filters and common operational actions.

5. Automation and rules

Main rules run in the database to keep consistency even with multiple connected clients. Triggers control auditing, soft delete and automatic updates of wallet balance and debt. 


-- Fines and payments
CREATE TRIGGER trg_apply_fine
AFTER INSERT ON fine
EXECUTE FUNCTION apply_fine_to_wallet();

CREATE TRIGGER trg_apply_fine_payment
AFTER INSERT ON fine_payment
EXECUTE FUNCTION apply_fine_payment();

-- Generic audit
CREATE TRIGGER audit_app_user
AFTER INSERT OR UPDATE OR DELETE ON app_user
EXECUTE FUNCTION audit_log_generic();

In addition, updates are blocked on deleted records, pending fines can be canceled in specific cases, and indexes accelerate status/date filtering.

7. SQL and queries

The SQL console enables safe queries (read-only mode), result export and ready examples to speed up analysis. The focus is enabling operational investigation without breaking database rules. 


-- Quick read example
SELECT * FROM app_user_active LIMIT 10;

-- Pending fines per citizen
SELECT citizen_id, COUNT(*) AS pendentes, SUM(amount) AS total
FROM fine
WHERE status = 'pending'
GROUP BY citizen_id;

8. Conclusion

SmartCityOS delivers a solid relational base, automated rules and an interface ready for operation. The result is an end-to-end system that demonstrates SQL mastery, data architecture and user experience.
